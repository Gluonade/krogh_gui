\section{Scope and intent}
This document is the detailed technical reference for the current Krogh GUI project state. It is written so that a natural scientist can reconstruct what the software computes, how it computes it, and where this logic is implemented in Python.

The documentation is intentionally explicit in five dimensions:
\begin{enumerate}
    \item mathematical formulation and equation meaning,
    \item numerical method and solver workflow,
    \item Python implementation mapping (file, function, inputs, outputs, call context),
    \item GUI structure and user actions,
    \item output files and generated artifacts (what, where, when).
\end{enumerate}

\section{Mathematical model}
\subsection{Geometry and state variables}
The model uses a Krogh-cylinder representation:
\begin{itemize}
    \item capillary radius: $R_{cap}=5\times10^{-4}$ cm,
    \item tissue radius: $R_{tis}\in\{30,50,100\}\times10^{-4}$ cm (default base geometry: $100\times10^{-4}$ cm),
    \item capillary length: $L_{cap}=0.1$ cm.
\end{itemize}

In the GUI, the base tissue radius is selectable as 30, 50, or 100 $\mu$m. The selected value is applied globally to the forward-model geometry (radial grid, radial averaging weights, and all single/series simulations), while the diagnostic radius-sensitivity comparison remains available as an additional independent reconstruction analysis.

Unknown fields:
\begin{itemize}
    \item capillary oxygen partial pressure along axis: $P_c(z)$,
    \item tissue oxygen field: $P(z,r)$.
\end{itemize}

\subsection{Blood-gas correction of oxygen affinity}
The code computes an effective hemoglobin half-saturation pressure:
\[
P_{50,eff} = P_{50}\cdot10^{\Delta},
\quad
\Delta = \beta_{pH}(pH-PH_{ref}) + \beta_{CO_2}\log_{10}(pCO_2/PCO2_{ref}) + \beta_T(T-T_{ref}).
\]

Constants in \texttt{krogh\_app/constants.py}:
\begin{itemize}
    \item $P_{50}=26.8$ mmHg,
    \item $\beta_{pH}=-0.48$,
    \item $\beta_{CO_2}=0.06$,
    \item $\beta_T=0.024$,
    \item $PH_{ref}=7.40$, $PCO2_{ref}=40$ mmHg, $T_{ref}=37^\circ$C.
\end{itemize}

Interpretation in plain language: this step shifts the oxygen dissociation curve depending on acidity, CO$_2$, and temperature. A shifted curve changes how strongly pressure drops for a given extraction level.

\subsection{Hemoglobin saturation and differential capacity}
Hill saturation:
\[
S(P)=\frac{P^n}{P^n+P_{50,eff}^n}, \quad n=2.7.
\]

Differential oxygen capacity used in the axial capillary ODE:
\[
\frac{dC}{dP}=C_{Hb}\frac{dS}{dP}+\alpha,
\]
with $C_{Hb}=0.20$ and $\alpha=3\times10^{-5}$ in current implementation units.

Meaning: the same extraction flux causes a larger pressure drop where $dC/dP$ is small (flatter dissociation segment), and a smaller pressure drop where $dC/dP$ is large.

\subsection{Radial Krogh-Erlang profile}
At fixed $z$ and effective metabolic demand $M$, the radial approximation is
\[
P(r)=P_c + \frac{M}{4K}(r^2-R_{cap}^2) - \frac{MR_{tis}^2}{2K}\ln\left(\frac{r}{R_{cap}}\right),
\]
with diffusion parameter $K=2\times10^{-9}$.

This profile is used as a direct local estimate and as initialization for the 2D iterative field solve.

\subsection{Oxygen-dependent metabolic consumption}
The code uses a Michaelis-Menten-like saturation law with a nonzero floor:
\[
M(P)=M_{max}\left[f_{min} + (1-f_{min})\frac{P}{P+P_{half}}\right],
\]
where $f_{min}=0.05$ and $P_{half}$ is the mitochondrial half-saturation parameter (fit parameter in inverse mode).

Meaning in plain terms: metabolism decreases at low oxygen but never to exact zero. This avoids non-physiological collapse and improves numerical stability.

\subsection{Capillary axial transport equation}
The axial capillary equation solved numerically is
\[
\frac{dP_c}{dz}=-\frac{M_{eff}(z)\,\pi R_{tis}^2}{Q_{local}\,dC/dP},
\]
with
\[
Q_{local}=\max(\text{perfusion\_factor}\cdot Q_{flow},10^{-12}).
\]

The $M_{eff}$ term is obtained from iterative coupling to tissue oxygenation (without axial tissue diffusion) or from the 2D coupled field (with axial tissue diffusion).

\section{Numerical methods and algorithm workflow}
\subsection{Discretization}
Default mesh (from \texttt{krogh\_app/constants.py}):
\begin{itemize}
    \item axial points: $NZ=80$,
    \item radial points: $NR=80$,
    \item \texttt{z\_eval = linspace(0, L\_cap, NZ)},
    \item \texttt{r\_vec = linspace(R\_cap, R\_tis, NR)}.
\end{itemize}
Radial averaging uses area-aware weighting \texttt{radial\_weights}.

\subsection{Capillary ODE integration}
Library method: \texttt{scipy.integrate.solve\_ivp(..., method="RK45")}
with defaults:
\begin{itemize}
    \item \texttt{rtol = 1e-9},
    \item \texttt{atol = 1e-11},
    \item \texttt{max\_step = dz}.
\end{itemize}

\subsection{2D tissue field with axial diffusion}
If axial diffusion is enabled, \texttt{solve\_tissue\_field\_with\_axial\_diffusion(...)} iterates a finite-difference update over the $(z,r)$ grid with:
\begin{itemize}
    \item inner iteration cap: \texttt{AXIAL\_DIFFUSION\_MAX\_ITER = 140},
    \item convergence criterion: \texttt{max(abs(P - P\_old)) < 1e-5},
    \item relaxation factor: \texttt{AXIAL\_DIFFUSION\_RELAX = 0.45}.
\end{itemize}

Boundary treatment in code:
\begin{itemize}
    \item capillary boundary fixed to current \texttt{P\_c\_axial},
    \item no-flux outer radial boundary via copy of neighbor node,
    \item axial end handling by nearest-neighbor copy.
\end{itemize}

\subsection{Outer capillary-tissue coupling}
\texttt{solve\_axial\_capillary\_po2(...)} alternates between tissue-field update and capillary ODE re-integration:
\begin{enumerate}
    \item solve tissue field from current \texttt{P\_c\_axial},
    \item compute slice-wise metabolic demand from tissue field,
    \item re-solve capillary ODE with interpolated demand,
    \item stop if capillary-profile change is below \texttt{AXIAL\_COUPLING\_TOL = 1e-4} or max iterations reached.
\end{enumerate}

This operator-splitting approach is transparent and robust for the intended workflow.

\subsection{Single-case output metrics}
\texttt{run\_single\_case(...)} computes and returns:
\begin{itemize}
    \item $P_{venous}$ (capillary terminal PO$_2$),
    \item $P_{tissue,min}$,
    \item $P_{tissue,p05}$ (5th percentile of all tissue nodes),
    \item hypoxic fractions below 1, 5, 10 mmHg,
    \item fractions above absolute thresholds (default 100 and 200 mmHg, plus optional extras),
    \item fractions above relative thresholds (defaults 90, 50, 30\% of chosen reference),
    \item relative reference key and value (\texttt{inlet} or \texttt{tissue\_max}),
    \item sensor-like average PO$_2$ (weighted radial, then axial mean),
    \item arterial and venous saturations,
    \item flow in nL/s.
\end{itemize}

\section{Inverse reconstruction mathematics}
\subsection{Problem definition}
Given measured/entered values from diagnostics:
\begin{itemize}
    \item blood-gas PO$_2$, pCO$_2$, pH, temperature,
    \item sensor target PO$_2$,
    \item optional venous saturation (converted to venous PO$_2$ target),
\end{itemize}
fit a plausible tuple
\[
(\text{perfusion\_factor}, P_{half}, \text{metabolic\_rate\_rel})
\]
that reproduces sensor and venous targets.

\subsection{Optimization strategy in current code}
Main API: \texttt{KroghReconstructor.fit\_joint\_parameters(...)} in \path{krogh_app/reconstruction.py}.

Current default strategy: \texttt{search\_strategy="optimized"}.

High-level workflow:
\begin{enumerate}
    \item Build geometric candidate grids (perfusion, $P_{half}$, metabolic factor).
    \item Screening pass to keep promising candidates.
    \item Local optimization around best screening seed.
    \item Probe neighborhood near best candidate.
    \item Post-fit two-stage 1D \texttt{brentq} refinement to enforce sensor target:
    \begin{itemize}
        \item stage 1: metabolic-rate refinement first,
        \item stage 2: perfusion refinement only if needed.
    \end{itemize}
    \item Attach uncertainty and identifiability summaries.
\end{enumerate}

Alternative mode: \texttt{legacy\_grid} (larger exhaustive candidate evaluation).

\subsection{Uncertainty and identifiability additions}
Recent extensions implemented in \path{krogh_app/reconstruction.py}:
\begin{itemize}
    \item \texttt{\_bootstrap\_uncertainty(...)}: repeated perturbed-target refits,
    \item \texttt{\_estimate\_identifiability(...)}: near-optimal set conditioning summary,
    \item fit warning logic includes large sensor/venous mismatch and wide uncertainty indicators.
\end{itemize}

\subsection{Radius-conditioned reconstructions and hidden burden}
During reconstruction plot-data construction:
\begin{itemize}
    \item hidden hypoxic burden map includes thresholds below 1, 5, 10, 15, 20 mmHg,
    \item radius scenarios are generated for 30, 50, and 100 \textmu m assumptions,
    \item summary text reports radius sensitivity and assumption-dependence.
\end{itemize}

\section{Diagnostic model mathematics}
The diagnostic layer in \path{oxygenation_diagnostic_mvp.py} uses explicit feature transforms.

\subsection{Feature risks}
Each major feature is mapped by a sigmoid or derived transform into $[0,1]$:
\begin{itemize}
    \item low blood-gas PO$_2$ risk,
    \item low tissue/sensor PO$_2$ risk,
    \item acidosis risk,
    \item hypercapnia risk,
    \item anemia risk,
    \item low venous saturation risk,
    \item temperature stress risk,
    \item arterial-to-sensor gap risk.
\end{itemize}

\subsection{Weighted risk score}
Current weight set in code:
\[
R = 0.22R_{po2} + 0.24R_{sensor} + 0.08R_{gap} + 0.12R_{acid} + 0.10R_{CO2} + 0.08R_{Hb} + 0.06R_{venous} + 0.02R_T.
\]
Score is clamped to $[0,1]$ and then hard-floor overrides apply for very low oxygen values.

\subsection{State probabilities}
Five states are scored using Gaussian-like PO$_2$-band centers plus risk/gap modifiers, then normalized with softmax:
\begin{itemize}
    \item normoxia,
    \item intermediate oxygenation,
    \item low oxygenation approaching critical,
    \item hypoxia,
    \item profound hypoxia.
\end{itemize}

This is probabilistic, not a single hard-threshold classifier.

\section{GUI structure and controls}
Main class: \texttt{KroghGUI(tk.Tk)} with window defaults \texttt{1080x860} and minimum \texttt{980x720}.

\subsection{Tabs and purpose}
\begin{itemize}
    \item Inputs: single-case physiological parameters (including perfusion factor, relative metabolic rate, and selectable tissue radius) and threshold settings.
    \item Series: 1D/2D sweeps, plot mode selection, optional save bundle.
    \item Numerics: solver and bootstrap settings.
    \item Diagnostic: blood-gas plus sensor interpretation, reconstruction, report export, benchmark launch.
\end{itemize}

\subsection{Diagnostic tab controls (current implementation)}
Controls include:
\begin{itemize}
    \item entries for PO$_2$, pCO$_2$, pH, temperature, sensor PO$_2$, hemoglobin, venous saturation,
    \item threshold entries for yellow/orange/red risk cutoffs,
    \item radius mode combo: \texttt{all variants} or \texttt{selected radius only},
    \item selected radius combo: 30, 50, 100 \textmu m,
    \item buttons: use single-case values, run diagnostic, save template, save diagnostic report, save publication report, reconstruct Krogh cylinder, run reconstruction benchmark,
    \item checkbutton: auto-save independent 30/50/100 \textmu m reconstructions to \texttt{Diagnostic reports}.
\end{itemize}

\subsection{Numerics tab settings}
User-adjustable numeric fields exposed through constants-backed specs:
\begin{itemize}
    \item ODE: \texttt{ode\_rtol}, \texttt{ode\_atol}, \texttt{ode\_max\_step},
    \item tissue iteration: \texttt{axial\_diffusion\_max\_iter}, \texttt{axial\_diffusion\_tol},
    \item coupling iteration: \texttt{axial\_coupling\_max\_iter}, \texttt{axial\_coupling\_tol},
    \item reconstruction uncertainty: \texttt{bootstrap\_samples}.
\end{itemize}

\section{Reproducibility and interpretation notes}
\begin{itemize}
    \item A fit is model-conditional: it is the best explanation inside current geometry/physiology assumptions.
    \item Boundary hits and wide uncertainty indicate weaker parameter separability.
    \item Radius sensitivity should be interpreted as assumption dependence, not as direct morphological measurement.
    \item Solver settings can materially affect difficult edge cases; numerics checks should be used in sweep studies.
\end{itemize}
